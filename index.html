Mac平台下搭建github+hexo个人博客

发表于 2018-06-12

作为自己的第一篇博客，主要记录一下自己搭建github+hexo博客的过程，也激励自己要常写博客，常总结常记录！
一、Requirements
在安装Hexo之前需要先安装Node.js和Git

1. Node.js
用nvm来安装Node.js或者Node官网下载LTS版本安装，nvm的方式安装,输入以下命令安装nvm：
cURL:

$ curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.2/install.sh | bash
Wget:
$ wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.3

nvm安装好后，新开一个终端输入以下命令来安装Node.js(默认安装node最新版本，可指定版本)
$ nvm install stable

  1. Git
    使用Homebrew命令行安装git
    $ brew install git
    二、安装Hexo
    1. 输入以下命令安装hexo：
      $ sudo npm install -g hexo
    2. 本地新建一个目录/Users/wxh/Work/hexo,进入hexo目录下执行:
      $ hexo init
      $ hexo install
    3. 生成静态文件：g是generate缩写
      $ hexo g
    4. 启动本地服务器：s是server的缩写
      $ hexo s
      此时浏览器中输入http://localhost:4000，即可看到效果
      三、关联github
    5. 在自己的github上新建一个仓库，名为yourname.github.io,yourname是github的用户名（我的是wangxianh.github.io），这个仓库命名规则固定不变。如果之前没有生成过ssh的key，需要新建一对ssh的key,并将id_rsa.pub添加到github,这里参考https://help.github.com/articles/adding-a-new-ssh-key-to-your-github-account/。
    6. 编辑本地hexo根目录下的_config.yml文件，找到deploy关键字，然后添加如下三行：
      deploy:
      type: git
      repository: git@github.com:yourname/yourname.github.io.git
      branch: master
      注意：每个关键字的冒号后面都有个空格
    7. 这样就关联好了github,接下来需要进入hexo目录下执行以下命令：
      1). $ hexo clean #清除缓存db.json和public文件夹
      2). $ hexo g #生成md文件的静态页面，md文件在hexo/source/_posts目录下，新写的md文件复制到此即可
      3). $ hexo d #将public文件夹中的内容推送到git仓库
      此时浏览器中输入https://yourname.github.io就能看到自己博客啦!
      注意：
      1）输入 hexo d可能出现如下问题：ERROR Deployer not found: git
      解决方案：终端输入：$ npm install –save hexo-deployer-git
      2) 输入hexo d推送到git时可能出现Permission denied(publickey)
      原因：没有将公钥添加到github
      解决方案：参考https://help.github.com/articles/adding-a-new-ssh-key-to-your-github-account/
      四、关联域名
      可以在阿里云上购买一个域名，在阿里云管理控制台找到购买的域名（注意域名状态审核中无法访问），添加两条域名解析规则：

这两条解析可以保证你的域名带不带www都能正常解析。
然后进入hexo/themes/landscape/source目录下新建CNAME文件(没有后缀)，内容为自己的域名（我的是hallow.top).
$ cd your-hexo/themes/landscape/source #到达指定目录
$ sudo vi CNAME #新建CNAME文件，输入自己的域名，:wq!保存退出
注：这个CNAME文件必不可少，每次运行generate的时候会把这个路径下的东西copy到public下.
之后在终端（hexo目录下）输入以下命令运行:
$ hexo clean
$ hexo g
$ hexo d
此时，输入我们的域名（hallow.top）就可以看到博客了。
五、更改博客主题
参考：http://theme-next.iissnan.com/getting-started.html#stable
下载next主题
在终端定位到hexo目录下，使用git下载主题：
$ cd your-hexo
$ git clone https://github.com/iissnan/hexo-theme-next themes/next
启用主题：
打开hexo根目录下的_config.yml(称为站点配置文件)，找到theme字段，将其值改为next
拷贝CNAME文件：
$ cp themes/landscape/source/CNAME themes/next/source #在hexo目录下将原主题中的CNAME拷贝到next主题的source文件下主题定制：
在站点配置文件中（hexo根目录下的_config.yml文件）：

  1. 设置作者昵称： 找到author关键字，设置autour为你的昵称
  2. 站点描述：找到description关键字，设置description为你的站点描述，可以是个性签名
  3. 选择Scheme: 搜索 scheme 关键字。 你会看到有三行 scheme 的配置，将你需用启用的 scheme 前面注释 # 去除即可，一般选择scheme: Pisces
  4. 设置语言：将 language 设置成你所需要的语言。建议明确设置你所需要的语言，例如选用简体中文，配置如下：
    language: zh-Hans

在主题配置文件中（hexo/themes/next目录下的_config.yml文件）：

  1. 设置菜单
    （1）设定菜单内容，对应的字段是 menu。 将你需用启用的 菜单前面注释 # 去除即可。
    注：若你的站点运行在子目录中，请将链接前缀的 / 去掉
    （2）设置菜单项的显示文本。以简体中文为例，若你需要添加一个菜单项，比如 something。那么就需要修改简体中文对应的翻译文件 languages/zh-Hans.yml，在 menu 字段下添加一项：
    menu:
    something: 有料
    （3）设定菜单项的图标
  2. 设置侧栏
    （1）设置侧栏的位置，修改 sidebar.position 的值，支持的选项有：
        left - 靠左放置
    right - 靠右放置
设置为靠左放置：
sidebar:
    position: left
    （2）设置侧栏的显示时机，修改 sidebar.display 的值
        post - 默认行为，在文章页面（拥有目录列表）时显示
    always - 在所有页面中都显示
    hide - 在所有页面中都隐藏（可以手动展开）
    remove - 完全移除
设置在所有页面中显示：
sidebar:
    display: always
  3. 设置头像，修改字段 avatar， 值设置成头像的链接地址
    avatar: /images/touxiang.png #将头像图片放置到source/images/目录下
