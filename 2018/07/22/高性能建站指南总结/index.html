高性能建站指南总结

发表于 2018-07-22 | 分类于 前端 | | 次阅读

前言

网站性能对前端工程师来说非常重要，书中总结了14条规则来提升网站性能，以下是自己的读书笔记。

1. 减少HTTP请求

  • 使用图片地图在一个图片上关联多个URL
  • 使用CSS Sprites将多幅图片合并为一幅单独的图片。使用background-image 和 background-position属性将HTML元素放置到背景图片中期望的位置。
  • 内联图片：通过使用data:URL模式可以在web页面中包含图片而不需要额外的http请求。
  • 合并脚本和样式表

2. 使用CDN(内容发布网络)

CDN是一组分布在不同地理位置的Web服务器。用于发布静态内容，包括图片、样式表、脚本和Flash。CDN是将源站内容分发至最接近用户的节点，使用户可以就近获取所需内容，提高用户访问的响应度和成功率。
CDN网络相当于是在用户和服务器之间增加Cache层，通过接管DNS的实现来将用户的请求引导到Cache上获得源服务器上的数据。

3. 添加Expires头

Web服务器使用Expires告诉客户端可以使用一个组件的缓存，直到指定的时间为止。
Expires缺点：

  • 要求服务器和客户端的时钟严格同步
  • 过期日期需要经常检查，一旦过期需要在服务器配置中提供一个新的日期

HTTP1.1中引入了Cache-Control来克服Expires的限制，Cache-Control使用max-age来指定组件缓存时间。如果两者同时使用，max-age会覆盖Expires.

4. 压缩组件

使用gzip压缩HTTP响应包来减少网络响应时间。压缩内容包括：HTML文档、脚本、样式表。
HTTP1.1开始，Web客户端可以通过HTTP请求中的Accept-Encoding来表示对压缩的支持.

Web服务器通过响应中的Content-Encoding来通知Web客户端。

5. 将样式表放在顶部

将样式表放在顶部的HEAD中可以避免白屏

6. 将脚本放在底部

7. 避免CSS表达式

8. 使用外部javascript和css

9. 减少DNS查找

10. 精简javascript

11. 避免重定向

12. 删除重复脚本

13. 配置ETag

14. 使Ajax可缓存
