Git基本操作

发表于 2018-06-19 | 分类于 前端 | | 次阅读

前言

自己在学习git的时候总结了一些git的基本操作命令，可以方便查阅。参考阮一峰的Git教程

一、基本步骤

  1. git init :初始化一个git仓库
  2. git add <filename> : 将工作区（本地文件夹）文件添加到版本库的暂存区（git仓库）
  3. git commit ：将暂存区内容提交到本地的master分支，HEAD指向master
  4. git remote add origin git@github.com:wangxianh/test.git : 关联远程库，只需关联一次。添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。
  5. git push -u origin master : 将本地库的所有内容推送到远程仓库
    由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令：git push origin master。

二、常用命令

  1. git status ：随时掌握工作区状态
  2. git diff ： 如果git status告诉你有文件被修改过，用git diff可以查看修改内容。
  3. git log ：显示从最近到最远的提交日志
  4. git reset --hard commit_id ：版本回退，回退到指定的commit_id，回退前，可以使用git log查看提交历史，以便确定回退到哪个版本。要重返未来，可以使用git relog查看命令历史(查看commit_id)，以便确定回到未来的哪个版本。
  5. git reset --hard HEAD^ ：回退到上一个版本, HEAD^^上上一个版本
  6. git checkout -- <filename> ：把在文件区的修改全部撤销，注意这里–，没有–,则代表切 换分支git branch dev
  7. git reset HEAD <filename> ：把暂存区的修改撤销掉，重新放回工作区。 git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区。当我们用HEAD时，表示最新的版本
  8. rm <filename> ：删除工作区的文件，或者直接在文件管理器中删除
  9. git rm <filename>:删除版本库中的文件，如果是文件夹用-r : git rm -r
  10. git clone git@github.com:wangxianh/test.git：从远程仓库克隆到本地库
  11. git remote -v ：查看远程库信息

三、分支管理

  1. git branch <name> ：创建分支
  2. git checkout <name>：切换分支
  3. git checkout -b dev： 创建分支dev并切换到dev分支，等价于git branch dev(创建dev分支)和git checkout dev（切换到dev）两个命令
  4. git branch：查看当前所有分支，当前分支前面会带有*号
  5. git merge <name>：合并分支，采用的是 fast forward模式合并
  6. git branch -d <name>：删除分支
  7. git merge --no--ff “no-ff合并” dev: 准备合并dev分支，请注意–no-ff参数，表示禁用Fast forward . 合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。
  8. git stash： 把当前工作现场“储藏”起来，等以后恢复现场后继续工作。 当手头工作没有完成时，先把工作现场git stash一下，然后去修复bug，修复后，再git stash pop，回到工作现场。
  9. git branch -D <name>：强行删除一个没有被合并过的分支

四、版本回退

  1. git reflog:查看所有历史修改，根据id可以回退到任意版本
  2. git reset --hard id: 版本回退，id版本号（通过git reflog查看）, 上一个版本就是HEAD^，上上一个版本就是HEAD^^

五、多人协作

多人协作的工作模式通常是这样：

  1. 首先，可以试图用git push origin branch-name推送自己的修改；
  2. 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；
  3. 如果合并有冲突，则解决冲突，并在本地提交；
  4. 没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功！
    如果git pull提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream branch-name origin/branch-name。

六、工作区和暂存区

工作区：计算机本地文件夹
暂存区：使用git init命令后会初始化一个git仓库，工作区下出现一个隐藏目录.git，这个不算工作区，而是Git的版本库

前面讲了我们把文件往Git版本库里添加的时候，是分两步执行的：
第一步：用git add把文件添加进去，实际上就是把文件修改添加到暂存区；
第二步：用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。
因为我们创建Git版本库时，Git自动为我们创建了唯一一个master分支，所以，现在，git commit就是往master分支上提交更改。可以简单理解为，需要提交的文件修改通通放到暂存区，然后，一次性提交暂存区的所有修改。
